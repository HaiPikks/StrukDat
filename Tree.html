Tree

Struktur data Tree mirip dengan Linked List karena setiap node berisi data dan dapat dihubungkan ke node lain.

Tree disebut pohon karena bentuknya bercabang seperti pohon.

Kenapa Tree Berguna?

  • Data hierarkis: sistem file, model organisasi
  • Basis data: pengambilan data lebih cepat
  • Tabel perutean: meroute data dalam algoritma jaringan
  • Pengurutan & pencarian: mencari/mengurutkan data
  • Antrian prioritas: sering memakai heap (bentuk pohon)

Jenis-jenis Pohon

  • Binary Trees: tiap simpul maksimal punya 2 anak (kiri dan kanan).
  • Binary Search Trees (BST): kiri < root < kanan.
  • AVL Trees: BST yang diseimbangkan; selisih tinggi kiri-kanan maksimal 1.

Tree vs Array & Linked List

  • Array: cepat akses elemen, tapi insert/hapus perlu pergeseran memori.
  • Linked List: cepat insert/hapus, tapi akses elemen perlu penelusuran.
  • Tree: cepat akses node dan cepat insert/hapus tanpa pergeseran besar.

Binary Trees

Binary Tree adalah tree dengan maksimal dua anak untuk setiap simpul: anak kiri dan anak kanan.

Pembatasan ini membuat operasi seperti pencarian, insert, dan delete lebih mudah dan efisien.

Contoh implementasi: node saling terhubung ke left dan right.

Varian Binary Tree

  • Seimbang (balanced): selisih tinggi kiri-kanan maks 1.
  • Lengkap (complete): level terisi penuh dari kiri ke kanan.
  • Penuh (full): tiap simpul punya 0 atau 2 anak.
  • Sempurna (perfect): semua daun pada level sama; internal punya 2 anak.

Binary Search Tree (BST)

BST adalah binary tree dengan aturan khusus:

  • Semua nilai di subtree kiri < root
  • Semua nilai di subtree kanan > root

Contoh

Root 50, kiri 30 (20,40), kanan 70 (60,80).

        50
       /  \
     30    70
    / \    / \
   20 40  60 80

Sifat BST

Untuk setiap node: kiri < root < kanan.

Operasi Dasar BST

  1. Search (Pencarian): sama → ketemu, lebih kecil → kiri, lebih besar → kanan.
  2. Insertion (Penyisipan): ikuti aturan BST sampai menemukan posisi kosong.
  3. Deletion (Penghapusan): ada 3 kondisi (0 anak, 1 anak, 2 anak) lalu ganti sesuai strategi inorder successor/predecessor.

Traversal pada BST (Inorder)

Urutan traversal inorder: kiri → root → kanan. Hasilnya selalu urut naik.

Contoh: 20 30 40 50 60 70 80.

Kelebihan BST

  • Pencarian cepat
  • Insert cepat
  • Data otomatis terurut

Kekurangan BST

Jika tree tidak seimbang, BST bisa jadi lambat (mirip linked list).

Buat BST menggunakan Python untuk:

  1. Menambahkan data: 50, 30, 70, 20, 40, 60, 80
  2. Menampilkan inorder traversal
  3. Jelaskan kenapa hasilnya terurut
class Node:

    def __init__(self, data):

        self.data = data

        self.left = None

        self.right = None



def insert(root, data):

    if root is None:

        return Node(data)

    if data < root.data:

        root.left = insert(root.left, data)

    else:

        root.right = insert(root.right, data)

    return root



def inorder(root):

    if root:

        inorder(root.left)

        print(root.data, end=" ")

        inorder(root.right)



root = None

data = [50, 30, 70, 20, 40, 60, 80]

for i in data:

    root = insert(root, i)

print("Hasil inorder traversal:")

inorder(root)
